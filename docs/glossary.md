# Glossary

Domain vocabulary used throughout the Cappy codebase and documentation.
When in doubt, defer to this list.

| Term | Definition | Notes |
|------|------------|-------|
| Caregiver | Any adult authorized to participate in a family's medication coordination | Generic — covers parents, grandparents, sitters |
| Family | The unit of organization in Cappy: a small group of caregivers + their children | One family per user account in alpha |
| Admin | A caregiver with full permissions in a family, including managing other caregivers | Typically a parent |
| Read-only caregiver | A caregiver who can see history but not log doses | Typically a non-resident parent |
| Child | A person whose medication is being tracked in a family | We do not say "patient" |
| Dose event | A record that a child received a dose of medication at a specific time | "Dose" for short; never "administration" |
| Dose correction | A new dose event that supersedes a previous one due to caregiver error | Original is preserved with `superseded` status |
| NFC tag | The physical NTAG215 sticker affixed to a medication container | "Tag" for short |
| Tag UID | The unique hardware identifier of an NFC tag | Hex string from the chip |
| Tag binding | The server-side mapping from a tag UID to a family + medication | Set during tag registration |
| Quick-action screen | The screen the app opens to immediately after an NFC tap | The product's signature interaction |
| Invite code | A 6-digit code generated by an admin to add a caregiver to the family | Single use, expires in 24 hours |
| BAA | Business Associate Agreement | Required for any vendor that sees PHI |
| PHI | Protected Health Information | The data HIPAA cares about |
| Audit event | An immutable record in the audit log capturing a state change | One per state-changing API call |
| Idempotency key | A client-generated UUID that deduplicates retries | Required on every POST that creates a resource |
| Modular monolith | One deployable artifact with strict internal module boundaries | Our backend architecture per ADR-0001 |

## Words we don't use

| Avoid | Use instead | Why |
|-------|-------------|-----|
| Patient | Child | Avoids clinical framing |
| Administration | Dose | Plain language |
| User | Caregiver | More specific |
| Household | Family | Matches user mental model |
| Group | Family | Same |
| Medication event | Dose event | Plainer |
| Submit | Log | Action-oriented |
| Member | Caregiver | Specific |
