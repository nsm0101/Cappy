# Founder Feedback

Drop notes for the Orchestrator here. The Orchestrator reads this
folder at the start of every session and acknowledges new notes in
the daily briefing.

Format suggestion: one file per topic, named with date and slug.

Examples:
- `2026-05-15-prefer-shorter-tickets.md`
- `2026-05-22-rethink-onboarding-flow.md`
- `2026-06-01-feedback-from-design-partner-anna.md`

Notes are read but never modified by the Orchestrator. The Orchestrator
does not delete feedback files; the founder can archive or remove
them when no longer relevant.
